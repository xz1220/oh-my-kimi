# UltraQA

Use this for high-risk changes that need more than normal unit tests.

## Coverage Areas

- Acceptance criteria.
- Unit, integration, and end-to-end behavior.
- Error states and edge cases.
- Backward compatibility.
- Accessibility and responsive behavior for UI.
- Security and privacy risks.
- Documentation accuracy.

Prioritize real checks over checklist theater. Report exact evidence and gaps.
