# Wiki

Use this when turning working knowledge into durable documentation.

## Workflow

1. Identify the audience and maintenance owner.
2. Read existing docs to avoid duplication.
3. Write the smallest page that answers the recurring question.
4. Link to source files, commands, or external docs instead of copying large
   material.
5. Add "last reviewed" or version context when the content may drift.

Prefer clear indexes and short runbooks over long prose.
