# Release

Use this when preparing a GitHub release, package release, or docs launch.

## Checklist

1. Confirm intended version and scope.
2. Review git status, recent commits, and unreleased changes.
3. Run the full relevant validation suite.
4. Update changelog or release notes from actual changes.
5. Confirm docs and install instructions match the release.
6. Check license and attribution files if bundled third-party content changed.
7. Prepare tag and release notes, but do not push unless explicitly asked.
