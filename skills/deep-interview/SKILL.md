# Deep Interview

Use this when the user has a goal but the constraints, success criteria, risks,
or first implementation slice are unclear.

## Workflow

1. Restate the goal in one sentence and name the uncertainty that matters most.
2. Ask 1-3 targeted questions. Prefer questions that expose scope, audience,
   constraints, deadline, quality bar, or tradeoffs.
3. If the user is unavailable, make conservative assumptions and mark them.
4. Convert the result into:
   - Objective
   - Non-goals
   - Constraints
   - Acceptance criteria
   - First implementation slice
   - Validation plan

## Output

Keep the interview compact. Do not ask broad preference questions when a
reasonable default is obvious.
