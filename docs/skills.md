# Skills

Skills are installed into `~/.kimi/skills/<name>/SKILL.md` and can be invoked
inside Kimi with `/skill:<name>`.

## Core Execution

| Skill | Purpose |
|---|---|
| `deep-interview` | Clarify vague requests with targeted questions |
| `plan` | Create an implementation plan |
| `ralplan` | Convert work into verifiable stories |
| `ralph` | Implement, verify, review, fix, and repeat |
| `autopilot` | End-to-end delivery workflow |
| `team` | Coordinate Kimi subagents |

## Quality Gates

| Skill | Purpose |
|---|---|
| `verify` | Evidence-based completion check |
| `code-review` | Findings-first code review |
| `security-review` | Security-focused review |
| `tdd` | Test-driven development loop |
| `visual-verdict` | Frontend visual QA |
| `ai-slop-cleaner` | Cleanup AI-generated bloat and stale comments |
| `ultraqa` | High-rigor QA before release |

## Utility

| Skill | Purpose |
|---|---|

| `mcp-setup` | Add recommended MCP servers |
| `help` | Catalog help |

| `trace` | Causal tracing |
| `deep-dive` | Thorough investigation |
| `deepinit` | Repository onboarding |
| `release` | Release checklist |

| `skillify` | Turn workflows into skills |
